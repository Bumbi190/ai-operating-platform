## Vad som är kvar att göra

### 1. Verifiera Facebook-token (PRIO 1)

Andre nämnde att Facebook-access-token möjligen inte är kopplad eller är ogiltig.
Innan vi förlitar oss på att FB-postning funkar:

```bash
# Kolla i Vercel → Settings → Environment Variables att dessa finns:
FACEBOOK_PAGE_ACCESS_TOKEN
FACEBOOK_PAGE_ID
```

Kolla också `platform_tokens`-tabellen i Supabase:
```sql
select platform, expires_at, updated_at from platform_tokens;
```

Om Facebook saknas eller är utgången måste den genereras om via Meta for
Developers / Graph API Explorer. Long-lived page access tokens varar 60 dagar.

Relevant kod: `apps/web/lib/media/facebook.ts`, `apps/web/lib/media/token-store.ts`,
`apps/web/app/api/media/cron/refresh-tokens/route.ts`.

### 2. Testa hela publish-flödet med faktiskt innehåll i kön (PRIO 2)

Just nu returnerade test-anropet `"nothing_to_publish"` eftersom kön är tom.
För att verifiera att IG + FB postas SAMTIDIGT behöver vi:

1. Säkerställa att det finns ett `media_scripts`-row med:
   - `video_status = 'ready'`
   - `status = 'approved'`
   - `published_at IS NULL`
   - `generated_at` inom senaste 3h
2. Trigga manuellt: `select omnira_cron.call_vercel('/api/media/cron/publish');`
3. Verifiera resultat:
   ```sql
   select id, status_code, content::text, created
     from net._http_response
    order by created desc limit 1;
   ```
4. Förväntat svar: `{"status":"published","platforms":"Instagram & Facebook",...}`
5. Kolla själva flödena på IG + FB-sidorna

Logik finns i `apps/web/app/api/media/cron/publish/route.ts`:
- IG publiceras först (fatal om fail)
- FB publiceras sen (non-fatal, alert om fail)

### 3. Verifiera första schemalagda cron-körning (PRIO 3)

Första schemalagda jobb körs **idag ~19:20 svensk tid** (`omnira_step1_evening`,
schemat `20 17 * * *` UTC). Sen följer step2-4 + publish till 20:00 svensk tid.

Verifiera när du orkar:
```sql
select j.jobname, r.status, r.start_time, r.end_time, r.return_message
  from cron.job_run_details r
  join cron.job j on j.jobid = r.jobid
 where r.start_time > now() - interval '24 hours'
 order by r.start_time desc;
```

Alla 12 jobs ska få status `succeeded` enligt schemat. Om något får `failed`,
läs `return_message` + kolla motsvarande HTTP-svar i `net._http_response`.

```sql
-- Status på alla schemalagda jobs
select jobname, schedule, active from cron.job order by jobname;

-- Senaste 10 schemalagda körningarna
select j.jobname, r.status, r.start_time, r.end_time, r.return_message
  from cron.job_run_details r
  join cron.job j on j.jobid = r.jobid
 order by r.start_time desc limit 10;

-- Senaste 10 HTTP-svaren från pg_net → Vercel
select id, status_code, content::text, created
  from net._http_response
 order by created desc limit 10;

-- Manuellt trigga en endpoint

-- Ändra ett schema utan att ta bort jobbet
select cron.alter_job(
  job_id   := (select jobid from cron.job where jobname = 'omnira_publish_morning'),
  schedule := '30 9 * * *'
);

-- Pausa/återaktivera ett jobb
select cron.alter_job(job_id := ..., active := false);

-- Ta bort ett jobb helt
select cron.unschedule('omnira_step1_morning');
```

**Viktigt:** Alla scheman är i **UTC**. Stockholm är UTC+2 sommartid, UTC+1 vintertid.
